### 详细步骤

#### 步骤 1：扫描并导出数据（👤 人类操作）

1. **以管理员身份打开 WizTree**
   ```
   F:\coding\xianliao\WizTree\WizTree64.exe
   右键 → 以管理员身份运行
   ```

2. **扫描 C 盘**
   - 点击地址栏，输入 `C:`，按回车
   - 等待扫描完成（通常几秒钟）

3. **导出 CSV**
   - 点击菜单 "文件" → "导出"
   - 勾选选项：
     - ✅ 导出文件夹
     - ✅ 导出驱动器大小信息
     - ❌ 导出文件（不勾选，减小文件大小）
   - 保存到：`F:\coding\xianliao\skills\clean-c-drive\data\`

#### 步骤 2：AI 分析（🤖 自动）

在 Claude Code 中输入：

```
/clean-c-drive
```

#### 步骤 3：确认并执行清理（👤 人类操作）

1. 查看 AI 生成的清理报告
2. 选择清理级别（高/中/低）
3. 以管理员权限运行清理脚本：
   ```powershell
   powershell -ExecutionPolicy Bypass -File "F:\coding\xianliao\skills\clean-c-drive\clean_high.ps1"
   ```

#### 步骤 4：清理临时文件（可选）

清理完成后，AI 会询问是否删除扫描数据和清理脚本，选择"删除"保持目录整洁。

## 备份与回滚

### 备份机制

清理前会自动备份待清理的目录：

| 特性 | 说明 |
|------|------|
| **备份位置** | 自动选择非C盘剩余空间最大的驱动器 |
| **备份格式** | 小于1GB直接复制，大于1GB压缩 |
| **空间要求** | 至少需要 5GB 可用空间 |
| **备份记录** | 每次备份生成 `manifest.json` 记录原始路径 |

### 手动操作备份

```bash
# 查看备份驱动器
python backup.py drive

# 查看所有备份
python backup.py list

# 查看备份详情
python backup.py info --id backup_20260127_143052

# 手动回滚
python backup.py restore --id backup_20260127_143052

# 手动删除备份
python backup.py delete --id backup_20260127_143052

# 清理所有备份
python backup.py cleanup --all
```

### 回滚场景

如果清理后发现问题（如某些程序无法启动），可以一键回滚：

## 常见问题

### Q1: 为什么推荐使用 Opus 模型？

**A:** Opus 模型在处理复杂的文件系统操作、数据分析和多步骤任务时表现更好，能够：
- 更准确地理解清理需求
- 更可靠地执行文件操作
- 更好地处理异常情况

### Q2: 为什么管理员权限下 WizTree 可以命令行导出？

**A:** WizTree 使用 NTFS 的 MFT（主文件表）进行快速扫描，读取 MFT 需要管理员权限。命令行导出时也需要这个权限。

### Q3: 清理后会影响系统正常使用吗？

**A:** 不会。高优先级清理的都是缓存和临时文件：
- Windows 更新缓存：更新完成后不再需要
- 浏览器 IndexedDB：视频网站的播放缓存
- 临时文件：系统会自动重建

### Q4: 清理浏览器缓存会丢失数据吗？

**A:** 不会丢失重要数据：
- ✅ 保留：书签、密码、历史记录、登录状态
- ❌ 清理：IndexedDB（视频缓存）、GPUCache（着色器缓存）

### Q5: 多久清理一次比较合适？

**A:** 建议：
- 日常使用：每 2-4 周
- C 盘空间紧张：随时清理
- 安装大型软件前：先清理腾出空间

### Q6: 可以自定义清理规则吗？

**A:** 可以。编辑 `analyze.py` 中的 `CLEANABLE_PATTERNS`：

```python
CLEANABLE_PATTERNS = {
    "high": {
        "patterns": [
            {"pattern": "你的自定义模式", "name": "描述", "safe": True},
            ...
        ]
    }
}
```

### Q7: 清理完成后临时文件会自动删除吗？

**A:** 不会自动删除，AI 会询问你是否要删除：
- 扫描数据文件 (`data/*.csv`)
- 生成的清理脚本 (`clean_*.ps1`)

选择"删除"可以保持目录整洁。

### Q8: 备份会占用多少空间？

**A:** 备份空间取决于待清理目录的大小：
- 小于 1GB 的目录：直接复制，占用原始大小
- 大于等于 1GB 的目录：压缩备份，通常压缩率在 30-70%

备份会自动存放在非C盘剩余空间最大的驱动器。

### Q9: 备份会保留多久？

**A:** 备份不会自动过期，需要手动删除：
- 清理后验证系统正常 → AI 会询问是否删除备份
- 手动删除：`python backup.py delete --id <备份ID>`
- 清理所有备份：`python backup.py cleanup --all`

### Q10: 如果只有 C 盘怎么办？

**A:** 如果只有 C 盘或其他盘空间不足（小于 5GB），备份功能将无法使用。建议：
1. 先手动清理其他磁盘腾出空间
2. 或者添加一个外置存储设备
3. 或者选择跳过备份（风险自负）

## WizTree 命令行参数参考

```powershell
WizTree64.exe "C:" /export="output.csv" /admin=1 /exportfolders=1 /exportfiles=0 /exportmaxdepth=200
```

| 参数 | 说明 |
|------|------|
| `/admin=1` | 使用管理员模式（MFT 扫描） |
| `/export="file.csv"` | 导出到 CSV 文件 |
| `/exportfolders=1` | 导出文件夹 |
| `/exportfiles=0` | 不导出文件（减小文件大小） |
| `/exportdrivecapacity=1` | 导出磁盘容量信息 |
| `/sortby=2` | 按大小排序 |
| `/exportmaxdepth=200` | 最大文件夹深度（减小文件大小） |

**优化提示：** 设置 `/exportmaxdepth=200` 可以将导出文件从 160MB 减小到 10-20MB，大幅加快分析速度。
